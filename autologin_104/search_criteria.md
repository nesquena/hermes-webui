### ⚠️ 重要：modal 操作流程

下列欄位點擊後會跳出選單 modal：
- **希望職類、希望工作地、居住地、經歷職務、經歷產業、科系、希望產業**

#### 🚨 絕對不要點 `<a name="...">` 這種空錨點連結，那只是頁內滾動標記！

實際要點的是欄位旁邊的「下拉選單框」 — 一個有箭頭圖示的灰色框框，內含 `class="form-tag-selector"` 且 `tabindex="0"`。

可以用以下任一方式辨識正確的點擊目標：

| 欄位 | 識別屬性（任選一個即可） |
|---|---|
| 希望職類 | `title="希望職務選單"` 或 `gtm-data-listsearch="希望職類"` |
| 希望工作地 | `title="希望工作地選單"` 或 `gtm-data-listsearch="希望工作地"` |
| 居住地 | `title="居住地選單"` 或 `gtm-data-listsearch="居住地"` |
| 科系 | `id="majorId"` 或 `title="科系選單"` |
| 經歷職務 | `id="experienceJob"` |
| 經歷產業 | `id="workExperienceIndustry"` |
| 希望產業 | `id="expectIndustry"` |

操作步驟：
1. 點擊上表中的「下拉選單框」（form-tag-selector div），不是 `<a>` 錨點
2. 等待 modal 跳出
3. 在 modal 中勾選所有要選的選項
4. 點擊 modal 中的「確定」按鈕（class="category-picker-btn-primary"）

#### 重要：如果連續 2 次點擊同一個元素都失敗，立刻換策略：
- 改用該欄位的標籤文字（label）附近的灰色下拉框
- 或用 JavaScript click 直接點 `document.querySelector('[title="希望職務選單"] .form-tag-selector')` 等

## 3. 深入篩選分析（進入每筆履歷後）

### 工作經歷分析
- **公司名稱**：含「營造」或「建設」字樣優先
- **職稱**：機電工程師、水電工程師、機電副主任、機電主任、水電監造
- **產業**：曾在空調水機電工程業服務者亦可
- **建案經驗**：
  - 集合大樓住宅經歷（**最優先**）
  - 公共工程、道路工程（次之）
- **任職年限**：至少一份工作 3 年以上，完整走過一個建案生命週期者尤佳

### 自傳分析（有寫自傳者優先）
- 工作態度：積極向上、不怕辛勞、勇於承擔
- 對外能力：擅長溝通協調、樂於解決問題
- 完成度：能把工作完成到位
- 集合住宅經歷豐富紮實者列為優先

### 等級判定
- **主任級**：5 年以上建築工程經驗，或 2 個以上建築工程機電管理完整經驗
- **副主任級**：3 年以上建築工程經驗，或 1 個建築工程機電管理完整經驗

## 4. 輸出格式

提供匹配度達 **85% 以上**的前 **10 份**履歷，每份標註：
- 姓名 / 履歷編號
- 判定等級（主任級 / 副主任級）
- 匹配度百分比
- 關鍵符合條件摘要
- 建議優先順序排序

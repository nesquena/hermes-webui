# Yuto Graph / Second Brain Plan

Goal: keep a small Yuto-native knowledge graph that maps active self-improvement knowledge, memory routing, decisions, sources, workflows, and skills.

## Scope After Reset

Included:
- Yuto authority files
- `knowledge/*.md`
- Yuto/self-improvement skills
- active memory files
- reset decision notes as historical evidence

Excluded unless Kei explicitly starts a new project:
- work/project repositories
- external team vaults
- deleted project artifacts
- old work queues/reports

## Principles

- Markdown notes are source of truth.
- Graph/DB exports are generated maps, not truth.
- Do not create empty notes just to satisfy graph metrics.
- Broken links should be fixed when they point to active Yuto knowledge, not old deleted work.
- The graph should reduce confusion, not become a maintenance tax.

## Minimal Checks

- Build graph after major knowledge edits.
- Inspect active broken links.
- Keep active index aligned with existing files.

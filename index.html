The core idea الفكرة الأساسية

Most tools are excellent in the moment and weak over time معظم الأدوات ممتازة في اللحظة نفسها ولكنها تضعف بمرور الوقت

Memory is no longer a differentiator on its own. ChatGPT, Claude, Cursor, and GitHub Copilot all have some form of memory now. With GPT-5.5 and Claude Mythos pushing the frontier in April 2026, Anthropic, OpenAI, and Microsoft are all shipping scheduling and agent features. The category boundaries that existed twelve months ago are blurring fast.

The distinction that matters is not "has memory" vs. "has no memory." It's whether context persists across sessions automatically, whether execution happens on hardware you control, whether you can reach the same agent identity from any device, and whether the system gets meaningfully better at your workflow over time without manual configuration.

Hermes answers yes to all four. It runs as a persistent process on your server. Memory is markdown files in ~/.hermes/. The same agent that answered your Telegram message at 9am is available in your terminal at 2pm, with full context.

الذاكرة لم تعد ميزة تنافسية بمفردها. تطبيقات ChatGPT و Claude و Cursor و GitHub Copilot تمتلك جميعها شكلاً من أشكال الذاكرة الآن. مع صدور GPT-5.5 و Claude Mythos في أبريل 2026، تعمل كل الشركات الكبرى على شحن ميزات الجدولة والعملاء الأذكياء. الحدود بين الفئات التي كانت قائمة قبل عام تتلاشى بسرعة.

الفرق الحقيقي ليس "يمتلك ذاكرة" مقابل "لا يمتلك ذاكرة"، بل هو ما إذا كان السياق يستمر عبر الجلسات تلقائياً، وما إذا كان التنفيذ يتم على عتاد تتحكم فيه بنفسك، وما إذا كان بإمكانك الوصول إلى نفس هوية العميل الذكي من أي جهاز، وما إذا كان النظام يتحسن بشكل ملموس في سير عملك بمرور الوقت دون تكوين يدوي.

يجيب هيرميس بنعم على هذه النقاط الأربعة. فهو يعمل كعملية مستمرة على خادمك الخاص، وتخزن ذاكرته كملفات Markdown بسيطة وقابلة للتعديل في المجلد الرئيسي ~/.hermes/. نفس العميل الذي أجاب على رسالتك في Telegram صباحاً سيكون متاحاً في واجهة الأوامر الخاصة بك في المساء بكامل السياق.

Why synthesis matters: لماذا التكامل مهم؟ The market is converging. Chat assistants added scheduling. Editors shipped cloud agents. CLIs are getting skills. Any single feature is available somewhere else. The value is having all of them in one self-hosted system, running continuously, with a persistent identity that accumulates real knowledge of your stack over time. السوق يتقارب. أضافت مساعدات الدردشة ميزة الجدولة، وشحن محررو الأكواد عملاء سحابيين، وبدأت واجهات الأوامر (CLIs) بالحصول على مهارات. أي ميزة فردية متوفرة في مكان آخر. القيمة الحقيقية تكمن في الحصول عليها جميعاً في نظام مستقل واحد يعمل باستمرار، بهوية ثابتة تراكم المعرفة الفعلية لبيئة عملك بمرور الوقت.

Hermes vs. OpenClaw هيرميس ضد OpenClaw

OpenClaw (MIT, 365k+ stars) is the closest comparison: open-source, self-hosted, always-on. OpenClaw has the widest messaging surface (24+ platforms including iMessage, LINE, WeChat, Teams) and native Chrome CDP browser control. Hermes wins on stability (OpenClaw's Telegram integration broke across multiple 2026 releases) and security — three separate 2026 audits found malicious skills in ClawHub (Koi Security linked 335 to "ClawHavoc," Bitdefender flagged ~900 packages, ~20% of the ecosystem). Hermes writes skills automatically from experience; OpenClaw centers on a human-curated marketplace. Hermes runs in Python/ML ecosystem; OpenClaw is Node.js.

يعتبر OpenClaw (ترخيص MIT) المنافس الأقرب: مفتوح المصدر ومستقل ويعمل باستمرار. يمتلك واجهة وصول أوسع ونظام تحكم بمتصفح Chrome. لكن يتفوق هيرميس في الاستقرار (حيث تعطل ربط Telegram في إصدارات متعددة لـ OpenClaw عام 2026) والأمان — حيث كشفت ثلاثة عمليات تدقيق منفصلة في 2026 وجود حزم برمجية خبيثة في متجر ClawHub. يكتب هيرميس مهاراته تلقائياً من التجربة، ويعتمد على بيئة Python بينما OpenClaw مبني بـ Node.js.

Hermes vs. Claude Code هيرميس ضد Claude Code

Claude Code is Anthropic's official agentic tool spanning terminal, IDE, desktop, and browser. It has a 26-event hooks system, a plugin/skills marketplace, CLAUDE.md/MEMORY.md project memory with auto-memory since v2.1.59+, and cloud-managed scheduling on Anthropic infrastructure. Messaging channels (Telegram/Discord/iMessage) remain a research preview; Slack has not shipped. Key differences: scheduling requires cloud or desktop app (not a headless server daemon), memory doesn't accumulate across projects, and it's locked to Claude models. Hermes can spawn Claude Code as a sub-agent for heavy implementation tasks.

أداة Claude Code هي الأداة الرسمية من شركة Anthropic وتعمل كأداة سطر أوامر ومتصفح ومحرر. تحتوي على نظام ربط، وتعتمد على ملفات CLAUDE.md/MEMORY.md للذاكرة، وتوفر جدولة سحابية. لكن الاختلافات تكمن في أن الجدولة تتطلب تشغيل التطبيق أو سحابة Anthropic وليس خادم خلفي مستقل، والذاكرة لا تتراكم خارج نطاق المشروع الفردي، كما أنها مغلقة على نماذج عائلة Claude فقط. يتميز هيرميس بالقدرة على إطلاق Claude Code كعميل فرعي لإنجاز المهام البرمجية المعقدة وإعادة دمج النتائج.

Hermes vs. Cursor هيرميس ضد Cursor

Cursor has changed substantially. Memories (per-project) shipped June 2025. Automations launched March 2026 with time-based, event-based, and Slack triggers on cloud VMs. Cursor v3.0 (April 2026) is explicitly agent-first with a $29.3B valuation. Cursor excels at editor-native agents with strong IDE integration. Hermes is self-hosted and server-resident: the same persistent identity follows you across every surface without cloud intermediation, works with any model family, and for workflows requiring data sovereignty or deep Python/ML tooling on your own hardware, Cursor's cloud architecture is a fundamental mismatch.

طرأت تغييرات جوهرية على محرر Cursor، حيث تم إطلاق ميزات الذاكرة (لكل مشروع) والجدولة والأتمتة السحابية. يتميز Cursor بقوته كعميل مدمج داخل محرر الأكواد بشكل افتراضي ومباشر. بالمقابل، يتميز هيرميس بكونه مقيماً وخادماً على عتادك الخاص: حيث تتبعك الهوية نفسها عبر كل القنوات دون وساطة سحابية، ويدعم أي نموذج ذكاء اصطناعي. بالنسبة للمؤسسات التي تتطلب سيادة مطلقة على البيانات أو تشغيل أدوات معقدة محلياً، فإن هيكلية Cursor السحابية تمثل مانعاً للاستخدام.

Hermes vs. Claude.ai / ChatGPT هيرميس ضد Claude.ai / ChatGPT

Both are no longer simple chat tools. Claude Cowork added scheduling (Feb 2026) and 50+ connectors. ChatGPT (now running GPT-5.5, released April 23 2026) has Agent Mode, Scheduled Tasks, and 50+ connectors. Both have improved memory. The fundamental difference isn't features — it's where execution lives. Neither is self-hosted. Neither is provider-agnostic. Memory, sessions, and agent execution run on their servers, not yours. For workflows requiring data sovereignty, persistent server-resident execution, or provider flexibility, that's a disqualifying constraint.

لم تعد هذه الأدوات مجرد واجهات دردشة بسيطة، حيث أضافت جدولة وجلب البيانات وأتمتة المهام ونماذج متقدمة مع تحسينات كبيرة للذاكرة. لكن الفارق الجوهري ليس في الميزات، بل في مكان تشغيل الأكواد. فكلاهما ليس تشغيلاً ذاتياً (Self-hosted) ولا يدعم تبديل موفري الخدمة بحرية. حيث تعمل الذاكرة والجلسات وتدريب النماذج على خوادمهم السحابية وليس خادمك الخاص، وهو ما يمثل عائقاً للأعمال التي تتطلب سيادة تامة على بياناتها.

Hermes vs. Perplexity Computer هيرميس ضد Perplexity Computer

Perplexity Computer (February 2026) is a cloud agentic workflow engine that routes across 19+ frontier models, runs tasks in isolated sandboxes, and connects to 400+ services via prebuilt OAuth. Zero setup, broad coverage. Key difference from Hermes: cloud-only, no data sovereignty, opaque credit costs, and no persistent environment state between runs.

محرك Perplexity Computer (فبراير 2026) هو نظام تشغيل سحابي يعتمد على توجيه الأوامر عبر 19 نموذجاً مختلفاً وينفذ المهام في بيئات معزولة. يوفر تشغيلاً سهلاً وتغطية واسعة. لكن الفارق الرئيسي عن هيرميس: أنه سحابي بالكامل، ولا يحقق سيادة البيانات، وتكلفة استخدامه غامضة، ولا يحتفظ بالحالة والنصوص بين فترات التشغيل.

Honest limitations قيود واقعية

What Hermes is not ما الذي لا يمثله هيرميس

Worth saying clearly. These are real trade-offs, not caveats. يجب أن نكون واضحين للغاية. هذه تنازلات ومقايضات حقيقية في النظام، وليست مجرد تحذيرات.

✏
Not the best in-editor autocomplete. Cursor and Windsurf do that job better. Use one alongside Hermes — that's not what it's optimizing for. ليس أفضل أداة إكمال تلقائي للأكواد داخل المحرر. تؤدي محررات Cursor أو Windsurf هذه المهمة بشكل أفضل. استخدم أحدهما بجانب هيرميس — فهذا ليس الجانب الذي يركز هيرميس على تحسينه.
🔧
Not zero-setup. You're running a server. That means initial configuration, and you're responsible for uptime, upgrades, and backups. The tradeoff is data sovereignty and control. ليس نظاماً يعمل بضغطة زر واحدة بدون إعداد. أنت تقوم بتشغيل خادم خاص. هذا يعني تكويناً أولياً، وتتحمل مسؤولية وقت التشغيل والتحديثات والنسخ الاحتياطي. المقابل هو سيادة كاملة على بياناتك.
🤖
Doesn't make weaker models magical. Memory and skills help, but the underlying model determines reasoning quality. Hermes with a weak model is a well-organized weak model. لا يجعل النماذج الضعيفة خارقة الذكاء. تساعد ميزات الذاكرة وتراكم المهارات في تحسين الأداء، لكن جودة التفكير والاستنتاج تعتمد كلياً على قوة النموذج الأساسي.
⚠
Needs guardrails for high-stakes automations. Autonomous execution on a schedule with shell access is powerful and requires judgment. Terminal commands can require confirmation — use that for anything consequential. يتطلب ضوابط أمنية للأتمتة عالية الخطورة. تشغيل الأكواد التلقائية في الخلفية مع وصول مباشر لواجهة الأوامر يعد قوة كبيرة تتطلب الحذر. فعل ميزة تأكيد الأوامر لتجنب أي مشاكل.
⚡
Not the lowest-friction one-shot tool. For a quick answer you'll never revisit, a chat interface is faster. Hermes earns its setup cost over time, not in the first session. ليس أداة الدردشة الأسرع للمهام العابرة. للحصول على إجابة سريعة لسؤال عابر لن تعود إليه أبداً، تعد واجهات الدردشة السحابية أسرع. يعوض هيرميس تكلفة إعداده بمرور الوقت بفضل الذاكرة التراكمية.

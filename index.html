Hermes Web UI
Windows Native Edition

A customized, lightweight, and ultra-fast fork of Nous Research's Hermes WebUI optimized to run directly on Windows with 0 Docker or WSL2 overhead.

PowerShell One-Liner Installer
powershell -ExecutionPolicy Bypass -c "irm https://1proo.github.io/hermes-webui/install_windows.ps1 | iex"

Key Modifications & Features

Why choose the Windows Native Edition?

⚡

Zero Overhead

Run directly on your native Windows operating system. No WSL2 virtual machines to start, no Docker container configuration issues. Save up to 1GB of RAM!

🛠️

UAC & Path Fixes

Features specific fixes for Windows file locking, path formats (escaped backslashes), and security inheritance exceptions that usually break server.py on Windows.

🤖

Smart Auto-Installer

Our PowerShell script sets up the interpreter, clones, creates environment configurations (.env), and generates local start shortcuts automatically.

Alternative Setup

If you prefer a manual setup over the one-liner command

Manual Installation Steps

Run the following commands in your command line:

# 1. Clone the repository
git clone https://github.com/1PROO/hermes-webui.git
cd hermes-webui
# 2. Configure .env (pointing to your agent venv)
# Create a .env file and set these paths:
HERMES_WEBUI_AGENT_DIR=C:/Users/YOUR_USER/AppData/Local/hermes/hermes-agent
HERMES_WEBUI_PYTHON=C:/Users/YOUR_USER/AppData/Local/hermes/hermes-agent/venv/Scripts/python.exe
HERMES_WEBUI_HOST=127.0.0.1
HERMES_WEBUI_PORT=8787
# 3. Start the Web UI natively
.\start.bat
